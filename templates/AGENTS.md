# Modelos de Documentos — templates/

Esta pasta contém os arquivos de modelo padrão (templates) que servem de base para a geração automática dos documentos do projeto em cada fase do harness v6.

## 📄 Resumo dos Modelos

- [RAG-TEMPLATE.md](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/RAG-TEMPLATE.md) — Modelo padrão para criação de novos arquivos RAG (documentação de contexto), definindo o cabeçalho YAML obrigatório e as 7 seções estruturais.
- [PROMPT-PAGE-TEMPLATE.md](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/PROMPT-PAGE-TEMPLATE.md) — Molde de arquivo de prompt que guia as especificações visuais de telas ou fluxos de componentes CSS.
- [TASK-PROMPT-TEMPLATE.md](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/TASK-PROMPT-TEMPLATE.md) — Modelo padrão para estruturação de instruções e prompts de tarefas enviadas para agentes executores.
- [PRD-TEMPLATE.html](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/PRD-TEMPLATE.html) — Esqueleto HTML que padroniza o formato do documento de Requisitos de Produto (Product Requirements Document).
- [SPEC-TEMPLATE.html](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/SPEC-TEMPLATE.html) — Esqueleto HTML que padroniza a formatação do documento de Especificação Técnica de Arquitetura.
- [SPRINT-TEMPLATE.json](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/SPRINT-TEMPLATE.json) — Schema JSON base para estruturar a definição geral de escopo de sprints técnicas.
- [SPRINT-DETAIL-TEMPLATE.json](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/SPRINT-DETAIL-TEMPLATE.json) — Schema JSON para detalhamento minucioso de subtarefas, testes associados e critérios de aceitação.
- [CROSS-SPRINT-TEMPLATE.json](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/CROSS-SPRINT-TEMPLATE.json) — Arquivo modelo para gerenciar dependências e integrações técnicas entre múltiplos ciclos de sprints.
- [MILESTONE-TEMPLATE.json](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/MILESTONE-TEMPLATE.json) — Modelo JSON para mapeamento de metas, entregáveis macro e regras de portão dos checkpoints do projeto.
- [E2E-CHAIN-TEMPLATE.json](file:///home/kingdev/Documentos/Opencode_agents_v6/templates/E2E-CHAIN-TEMPLATE.json) — Configuração padrão para a cadeia de validação técnica integrada de ponta a ponta.
