Ajustar a integração do ai-jail no harness para que o opencode acesse corretamente `~/.config/opencode` e `~/.opencode` dentro dos projetos, documentar as mudanças, profissionalizar o README e fazer redesign completo da landing page, e criar a release v6.4.1 no GitHub.

### 2. `CHANGELOG.md` — Nova entrada v6.4.1
Adicionar no topo (após o header) uma seção:

```markdown
## [6.4.1] - 2026-07-15

### Fixed
- **ai-jail wrapper**: Corrigido `--private-home` → `--no-private-home` para permitir acesso a `~/.config/opencode` (agents, commands) e `~/.opencode` (plugins) dentro do sandbox
- **harness-workspace.ts**: Ferramenta agora lê `.harness/ai-jail.json` para gerar o `.ai-jail` dinamicamente

### Added
- **`.harness/ai-jail.json`**: Novo arquivo de configuração do ai-jail por projeto (rw_maps, masks, hide_dotdirs, no_private_home, no_docker)
- Documentação completa da integração ai-jail no README

### Changed
- Wrapper do opencode usa `--no-private-home` para preservar acesso ao config do opencode
- Configuração do `.ai-jail` atualizada com `private_home = false`
```
