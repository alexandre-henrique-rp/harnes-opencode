1. Objetivo

Este SPEC detalha o módulo user do sample-web-app. Cobre cadastro, busca, atualização e exclusão de clientes. Integra com ViaCEP para autocomplete. Compliance com LGPD. Personas: P-001 (lojista) e P-002 (cliente final) — ver PRD.html.

5. Contratos de API

POST /api/users — Cria user. Auth required. 10/min.
GET /api/users/:id — Busca user. Auth required. 100/min.
GET /api/users?cpf=X — Busca por CPF. Auth required. 100/min.
DELETE /api/users/:id — Soft delete (LGPD). Auth required. 10/min.

7. Componentes / Módulos

7.1 UserForm (frontend)

Form de cadastro. Campos: cpf, nome, email, cep, logradouro, bairro, cidade, uf. Ações: submit, clean. Integra ViaCEP no on-blur do cep.

7.2 userCreator service (backend)

Recebe input validado, criptografa CPF, persiste no DB. Audit log.

7.3 cpfValidator (lib)

Validação de dígitos verificadores. Pure function. Sem dependência externa.
