Por que mudar?

Engenharia Tradicional vs OpenCode Harness

Sem Harness

  • Backlog infinito e meses para entregar MVPs
  • "Depois eu faço os testes" (Débito Técnico)
  • Vulnerabilidades e falhas de LGPD em Produção
  • Equipes sobrecarregadas com trabalho braçal

Com OpenCode Harness

  • Software em Produção em poucos dias
  • TDD Obrigatório (Nenhum código sem teste)
  • Auditoria Jurídica Automática (DPO Agent)
  • Devs focam na estratégia, Agentes codificam

Seguranca

Sandbox ai-jail

Execucao isolada em nivel de kernel para proteger o sistema host durante a execucao de agentes.

Protecao Nativa do Sistema

O harness utiliza o ai-jail para isolar a execucao dos agentes usando bubblewrap (Linux) ou sandbox-exec (macOS). Comandos perigosos sao bloqueados automaticamente.

  • Bloqueio de comandos destrutivos (rm -rf /, fork bombs)
  • Mascaramento de arquivos sensiveis (.env, credentials)
  • Ocultacao de diretorios privados (.netrc, .kube)
  • Acesso preservado a config do opencode (agents, commands)
  • Configuracao customizavel por projeto via .harness/ai-jail.json

6 Fases Controladas

Cada fase possui um gate binario que precisa ser aprovado para avancar.

Fase 0

Briefing e Definicao

Analise do escopo, mapeamento de funcionalidades e aprovacao do desenvolvedor.

Aprovacao Manual
Fase 1

Documentacao e RAG

Construcao do mapa de contexto e catalogacao de documentacoes do projeto.

≥3 Docs Relevantes
Fase 2

Requisitos Tecnicos

Elaboracao do PRD e SPEC tecnica com auditoria de revisores.

PRD ≥ 80 & SPEC ≥ 85
Fase 3

Design e Componentizacao

Mapeamento de telas e integracao com Google Stitch para geracao de interfaces.

Score Estetico ≥ 70
Fase 4

Planejamento de Sprints

Decomposicao em tarefas gerenciaveis com 100% de cobertura da SPEC.

100% Cobertura SPEC
Fase 5

Build + Quality

Codificacao, testes TDD, varreduras de seguranca e conformidade LGPD.

Testes ≥ 85% & 0 Falhas
Regras

Principios Nao-Negociaveis

01

Single Responsibility

Cada agente resolve apenas um problema delimitado.

02

Defense in Depth

3 camadas de permissao protegem a execucao de comandos e arquivos.

03

Declarative State

Maquina de estados e fluxo declarados em contratos JSON auditaveis.

04

Lean Context

RAGs locais crescem organicamente sem inflar o contexto de prompt.

05

Audit Total

Toda chamada de ferramenta e gravada em logs append-only.

06

TDD Obrigatorio

Codigo sem teste automatizado correspondente e bloqueado.

07

API Docs Estrita

Funcoes publicas devem conter JSDoc/docstring com @param e @returns.

08

Simplicidade (YAGNI/KISS)

Abstracoes so a partir da terceira repeticao do mesmo padrao.

Resiliencia

Protocolo de Falhas

Classe Gatilho Acao Corretiva
transient Erro de rede, API fora do ar Auto-retry 3x com backoff (1s, 3s, 9s)
quality Score abaixo do limite Rework automatizado (limite 2x)
user-action Permissao ou ambiguidade Pausa e escalacao para o usuario
fatal Erro de compilacao ou JSON corrompido Halt imediato, correcao manual
